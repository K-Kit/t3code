# Source Root (`src/`)

## Purpose

Root module directory containing application entry point, core state management, WebSocket transport, routing setup, and top-level business logic. Serves as the composition layer that assembles components, hooks, and utilities into a runnable application.

## Key Files

| File               | Description                                                                                                           |
| ------------------ | --------------------------------------------------------------------------------------------------------------------- |
| `main.tsx`         | Application entry point; creates root React element, sets up router with history (hash for Electron, browser for web) |
| `router.ts`        | TanStack Router initialization with QueryClient and store providers                                                   |
| `routeTree.gen.ts` | Auto-generated route tree (do not edit; regenerated by TanStack Router plugin)                                        |
| `store.ts`         | Zustand store for app state (projects, threads, hydration); localStorage persistence with Effect schema validation    |
| `types.ts`         | Local TypeScript types: Thread, Message, Project, TextDelta, TextBlock, ApprovalRequest                               |
| `wsTransport.ts`   | Low-level WebSocket: connection lifecycle, reconnection, message buffering, push event caching                        |
| `wsNativeApi.ts`   | High-level WebSocket RPC wrapper; export subscribe functions (onServerWelcome, onServerConfigUpdated, etc.)           |
| `nativeApi.ts`     | NativeApi static methods extension (used in browser test mocking)                                                     |
| `env.ts`           | Runtime environment detection (isElectron, isDev)                                                                     |
| `branding.ts`      | App display name constant                                                                                             |

## For AI Agents

### Working In This Directory

1. **State lifecycle**: App loads → `main.tsx` → `router.ts` (creates QueryClient, store) → renders first route → `store.ts` hydrates from localStorage → routes mount components.

2. **WebSocket setup**: `wsNativeApi.ts` exports subscribe functions. Components call these in effects to sync server state. `wsTransport.ts` handles the low-level connection and reconnection.

3. **Adding new state**: Determine if it's:
   - **Persistent**: Add to `store.ts` (app-wide) or create `useLocalStorage` hook
   - **Transient**: Create Zustand store file (e.g., `newStateStore.ts`) or use React state in component
   - **Logic-heavy**: Extract to `*-logic.ts` and test independently

4. **Adding new business logic**: Create `descriptive-name-logic.ts`, write unit tests in `descriptive-name-logic.test.ts`, call from components.

### Testing Requirements

- All logic modules must have `*.test.ts` counterparts
- Tests use Vitest with Effect schema validation utilities
- Mock WebSocket via `wsTransport` mock (see `wsNativeApi.test.ts`)
- Run: `bun run test`

### Common Patterns

**Zustand Store (Transient)**

```typescript
// New file: useNewStateStore.ts
import { create } from "zustand";

export const useNewStateStore = create<State>((set) => ({
  // state...
}));
```

**Zustand Store with localStorage (Persistent)**

```typescript
// In store.ts, add to persistedState and sync logic
const [value, setValue] = useLocalStorage(key, defaultValue, schema);
```

**Extract Logic to \*-logic.ts**

```typescript
// composerLogic.ts
export function parseMentions(text: string): Mention[] { ... }

// ComposerPromptEditor.tsx
const mentions = parseMentions(editorText);
```

**Subscribe to WebSocket Events**

```typescript
// In component effect
onServerWelcome((payload) => {
  // Handle welcome
});

// Subscribe to provider updates
onServerProvidersUpdated((payload) => {
  // Update store
});
```
