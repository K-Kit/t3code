# Routes (`routes/`)

## Purpose

TanStack Router file-based routes. Each file in this directory is automatically converted to a route by the TanStack Router plugin (`@tanstack/router-plugin/vite`). Routes define URL patterns and the components that render for each pattern.

## Route Files

| File                  | Route             | Description                                                         |
| --------------------- | ----------------- | ------------------------------------------------------------------- |
| `__root.tsx`          | `/`               | Root layout; renders app shell (header, sidebar, main content area) |
| `_chat.tsx`           | `/chat`           | Chat layout group; renders sidebar + main chat view                 |
| `_chat.index.tsx`     | `/chat`           | No thread selected; shows welcome or empty state                    |
| `_chat.$threadId.tsx` | `/chat/:threadId` | Thread detail view; renders chat messages and composer              |
| `_chat.settings.tsx`  | `/chat/settings`  | App settings page                                                   |
| `_chat.skills.tsx`    | `/chat/skills`    | Skills management page                                              |

## TanStack Router Conventions

### File Naming

- **Layout groups** (underscore prefix): `_layout.tsx` creates a layout boundary without affecting the URL pattern
  - `_chat.tsx` — Layout for `/chat` and sub-routes
  - `__root.tsx` — Root layout (double underscore)

- **Dynamic segments** (dollar sign): `$paramName.tsx` creates a dynamic route parameter
  - `$threadId.tsx` matches `/chat/:threadId` and extracts `threadId` from search params
  - Access via `useParams()` hook: `const { threadId } = useParams({ from: '_chat.$threadId' })`

- **Index routes** (`.index`): `_layout.index.tsx` matches the exact path without additional segments
  - `_chat.index.tsx` matches `/chat` (no thread ID)

### Route Tree Generation

The file structure automatically generates `routeTree.gen.ts`. **Do not edit this file**; it is regenerated by the TanStack Router plugin on every build.

```
routes/
├── __root.tsx
└── _chat.tsx
    ├── _chat.index.tsx       → /chat
    ├── _chat.$threadId.tsx   → /chat/:threadId
    ├── _chat.settings.tsx    → /chat/settings
    └── _chat.skills.tsx      → /chat/skills
```

becomes:

```typescript

```

## For AI Agents

### Working In This Directory

1. **File structure**: Create new routes by adding new `.tsx` files in this directory. Naming convention determines the route URL automatically.

2. **Layout boundaries**: Use underscore prefix for layouts that don't change the URL. Example:

   ```
   _auth.tsx          → layout for /login, /signup, etc.
   _auth.login.tsx    → /login
   _auth.signup.tsx   → /signup
   ```

3. **Dynamic parameters**: Use dollar signs for dynamic segments. Access with `useParams()`.

   ```typescript
   // _chat.$threadId.tsx
   const { threadId } = useParams({ from: "_chat.$threadId" });
   ```

4. **Nested layouts**: Layouts can nest. Child routes inherit parent layouts.

   ```
   __root.tsx
   └── _chat.tsx (layout)
       ├── _chat.index.tsx
       └── _chat.$threadId.tsx (inherits _chat layout)
   ```

5. **Search parameters**: For optional query string params, define in route component:
   ```typescript
   function ChatPage() {
     const search = useSearch({ from: "_chat.$threadId" });
     // search.model, search.view, etc.
   }
   ```

### Testing Routes

- Routes themselves don't need unit tests; test the components they render
- Components rendered by routes are tested in `src/components/**/*.browser.tsx` with Playwright
- Use `router.navigate()` to test navigation flows

### Common Patterns

**Dynamic Thread Route**

```typescript
// _chat.$threadId.tsx
import { createFileRoute } from '@tanstack/react-router';
import ChatThreadView from '~/components/ChatThreadView';

export const Route = createFileRoute('/_chat/$threadId')({
  component: ChatThreadPage,
});

function ChatThreadPage() {
  const { threadId } = Route.useParams();
  return <ChatThreadView threadId={threadId} />;
}
```

**Layout with Nested Routes**

```typescript
// _chat.tsx
import { createFileRoute, Outlet } from '@tanstack/react-router';
import { Sidebar } from '~/components/Sidebar';
import { ChatView } from '~/components/ChatView';

export const Route = createFileRoute('/_chat')({
  component: ChatLayout,
});

function ChatLayout() {
  return (
    <div className="flex">
      <Sidebar />
      <main className="flex-1">
        <Outlet /> {/* Renders child routes: index, $threadId, settings, etc. */}
      </main>
    </div>
  );
}
```

**Search Parameters**

```typescript
// _chat.$threadId.tsx
import { useSearch } from '@tanstack/react-router';

function ChatThreadPage() {
  const { threadId } = useParams({ from: '_chat.$threadId' });
  const search = useSearch({ from: '_chat.$threadId' });

  const model = search.model ?? 'gpt-4'; // e.g., /chat/abc?model=gpt-4
  const view = search.view ?? 'default'; // e.g., /chat/abc?view=split

  return <ChatView threadId={threadId} model={model} view={view} />;
}
```

**Navigation with Router**

```typescript
// In a component:
const router = useRouter();

function ThreadLink({ threadId }) {
  return (
    <a
      onClick={() => router.navigate({ to: '/chat/$threadId', params: { threadId } })}
    >
      Open thread
    </a>
  );
}
```
